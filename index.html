1.Java的反射机制

一、Java反射机制概述

1）Relfection（反射）被视为是“动态语言”的关键，反射机制允许程序在执行期间借助Reflection API取得任何类的内部信息，并能直接操作任意对象的内部属性及方法
2）加载完类之后，在堆内存的方法区中就产生了一个Class类型的对象（一个类只有一个Class对象），这个对象包含了完整的类的结构信息。我们可以通过这个对象看到类的结构，这个对象就像一面镜子，透过这个镜子看到累的结构，所以形象地称之为反射
    正常方式：引入需要的“包类”名称 ---> 通过new实例化 ---> 取得实例化对象
    反射方式：实例化对象 --->getClass()方法（拿到上面的Class对象） --->得到完整的“包类”名称...

动态语言 VS 静态语言：
    动态语言：
        是一类在运行时可以改变其结构的语言：例如新的函数、对象、甚至代码都可以被引进。
        主要动态语言：
            C+，JavaScript、PHP、Python
    静态语言：
        运行时结构不可变的语言就是静态语言，比如Java、C、C++
        Java不是动态语言，但有一定的动态性，可以利用反射机制、字节码操作获得类似动态语言的特性，Java的动态性让编程更加灵活  
Java反射机制提供的功能：
    在运行时判断任意一个对象所属的类
    在运行时构造任意一个类的对象
    在运行时判断任意一个类所具有的成员变量和方法
    在运行时获取泛型信息
    在运行时调用任意一个对象的成员变量和方法
    在运行时处理注解
    生成动态代理
反射相关的主要API：
    java.lang.Class:代表一个类
    java.lang.reflect.Method:代表类的方法
    java.lang.reflect.Field:代表类的成员变量
    java.lang.reflect.Constructor:代表类的构造器

二、理解Class类并获取Class实例

Class类：
    在Object类中定义了 public final Clas getClass()方法，此方法被所有子类继承，返回值是一个Class类，此类是Java反射的源头
    对每个类而言，JRE都为其保留了一个不变的Class类型的对象，该Class类型对象保留了某个结构（不只是类，如class/interface/annoation/primitive type等）的有关信息

获取Class类实例的四种方法
    1) Class clazz = String.class
    2) Class clazz = "java.lang.String"
    3) Class clazz = Class.forName("java.lang.String)
    4) 使用类的加载器

三、类的加载与ClassLoader的理解

了解：类的加载过程：
    当程序使用某个类时，如果该类还未被加载到内存中，系统会通过如下三个步骤进行初始化：
        类的加载（Load）：
            将类的class文件读入内存，并为之创建一个java.lang.Class对象。此过程由类加载器完成
        类的链接（Link）：
            将类的二进制代码合并到JRE中
        类的初始化（Initialize）：
            JVM负责对类进行初始化

了解：类加载器的作用：
    类加载：将字节码文件加载到内存，生成一个代表这个类的java.lang.Class对象，作为类数据的访问入口
    类缓存：一旦某个类被加载到类加载器中，将缓存一段时间。JVM的垃圾回收机制可以回收这些Class对象

七、反射的应用：动态代理

代理设计模式的原理：
    使用一个代理，将对象包装起来，然后用该代理对象取代原始对象。人核对原始对象的调用都要通过代理
静态代理：
    特征是代理类和目标对象的类在编译期间就确定下来了，不利于程序的扩展。同时每一个代理类只能为一个接口服务。
    [参考：静态代理：https://blog.csdn.net/qq_38974634/article/details/81255599]
动态代理：
    客户通过代理类带调用其他对象，并且是在程序运行时根据需要动态创建目标类的代理对象。可以实现通过一个代理类完成全部的代理功能。
    [参考：动态代理：https://blog.csdn.net/qq_38974634/article/details/81256347]


动态代理与AOP：
    AOP：Aspect Orient Programming 面向切面编程
        AOP能让我们在不影响原有功能的前提下，为软件横向扩展功能。
    从前：
        多个代码段含有相同代码段的处理是，将相同代码段独立成函数。
        这样的弊端是，代码段和函数硬编码耦合了
    理想情况：
        固定代码段 + 动态可修改的代码段
    实现方式：
        动态代理
        AOP代理的方法：
            动态代理里增加的通用方法1（硬编码）
            回调目标对象的方法（动态）       
            动态代理增加的通用方法2（硬编码）

八、其他

什么是Java Bean：
    JavaBean是一种特殊、可重用的Java类。必须有无参数的构造器，所有属性都是private的，通过提供getter和setter来实现对成员属性的访问，需要被序列化并且实现了java.io.Serializable接口
    ![参考：https://www.cnblogs.com/jyroy/p/11102298.html]

jar包的一些事：
    [参考：https://zhuanlan.zhihu.com/p/82320492]
    jar包简介：
        jar包，Java Archive File，jar包主要对.class文件进行打包，由于.class字节码文件是平台无关的，所以jar包也是平台无关的。
    jar包内容说明：
        jar包与zip包十分相似，唯一区别是jar包中包含了一些说明文件，对jar包版本、搜索路径、Main方法入口等进行了说明。
        除了.MF说明文件、.class文件外，jar还能打包静态资源文件如.html、.css等项目所需的一切，无论是web应用还是底层狂框架，都能打成jar包
    jar包执行：
        有的jar包可以通过java -jar来执行。能执行是因为有main函数，说明文件提供了函数入口
    为什么要打jar包：
        将整个项目封装起来，同时提供清晰地说明文档，供人方便地调用
    jar包和war包的区别：
        war是一个可以直接运行的web模块，将war包放在tomcat根目录的webapps下，tomcat启动，这个包就会自动解压，相当于发布
        war包的内容jar包也能打，而且war包仅用于Web应用，因此主流应用目前都用jar替代war。war相对于jar的唯一优势在于，tomcat启动后会自动将war包解压，对外提供web服务，而jar包不行。
        操作方法为：将tomcat和war包打成一个jar包，放在容器里单独运行。

1.JDBC概述

一、Java中的数据存储技术

在Java中，数据库存取技术可分为如下几类：
    JDBC直接访问数据库
    JDO(Java Data Object)技术
    第三方O/R工具，如Hibernate、Mybatis
JDBC是Java访问数据库的基石，JDO、Hibernate、MyBatis等只是更好的封装了JDBC

二、JDBC介绍

JDBC是一个独立于特定数据库管理系统、通用的SQL数据库存取和操作的公共接口（一组API），定义了用来访问数据库的标准Java类库，运行模式为：
    数据库厂商提供JDBC驱动程序，Java程序员直接调用统一API，大大加快开发过程
JDBC本质：
    一组规范、接口

三、JDBC体系结构

JDBC接口包括两个层次：
    面向程序员的API：Java API，抽象接口，供应用程序开发人员使用
    面向数据库的API：Java Driver API，供开发商开发数据库驱动程序使用

3.使用PreparedStatement实现CRUD操作

一、操作和访问数据库

Statement：用于执行静态SQL语句，并返回他生成的结构
PreparedStatement：SQL语句被预编译并存储在此对象中，可以使用此对象多次高效地执行该语句
CallableStatement：用于执行SQL存储过程

二、使用Statement操作数据表的弊端

1）存在拼串操作，繁琐
2）存在SQL注入问题
    指利用某些系统没有对用户输入的数据进行充分的检查，而在用户输入数据中注入非法的SQL语句段或命令，从而利用系统的SQL引擎完成恶意行为的做法

三、PreparedStatement的使用

介绍：
    PreparedStatement接口是Statement的子接口，他表示一条预编译过的SQL语句
PreparedStatement VS Statement：
    1）前者可读性和可维护性更高
    2）前者能最大可能提高性能：
        DBServer会对预编译语句提供性能优化
    3）前者可以防止SQL注入

一、配置ssh免密登录

原理：
    1.首先在客户端C生成一对密钥 ssh-genkey -t rsa
    2.将公钥拷贝给服务端S ssh-copy-id 服务端S的ip（首次登陆需要密码）
    3.当C向S发送连接请求时（用户名+ip），C会用公钥进行加密，S进行解密，然后随机生成字符串给C，C将解密后的字符串发给S，二者完成互相确认身份，S允许C免密登录

1. shell脚本编程

一、基本用法

变量：
    命名：区分大小写
    变量赋值：
        字符串：var = 'apple'
        变量引用：var = "$apple"，双引号是强引用，得到变量值，单引号是弱引用
        命令引用：var = $(COMMAND) 或 `COMMAND`
    环境变量：声明：export name=VALUE
    只读变量：声明：readonly name=VALUE
    位置变量：
        $1，$2：传给脚本的第1/2个参数
        $0：命令本身，包括路径
        $*：传给脚本的所有参数，合为一个字符串
        $@：传给脚本的所有参数，每个参数为独立字符串
        $#：传给脚本的参数个数
    退出状态码变量：
        $?：进程执行后，使用该变量保存进程执行状态，0为成功，其他为异常

0. InLong介绍

Apache InLong（应龙）是一站式的海量数据集成框架，提供自动、安全、可靠和高性能的数据传输能力，方便业务构建基于流式的数据分析、建模和应用。 InLong 项目原名 TubeMQ ，专注于高性能、低成本的消息队列服务。为了进一步释放 TubeMQ 周边的生态能力，我们将项目升级为 InLong，专注打造一站式海量数据集成框架。 Apache InLong 依托 10 万亿级别的数据接入和处理能力，整合了数据采集、汇聚、存储、分拣数据处理全流程，拥有简单易用、灵活扩展、稳定可靠等特性。 该项目最初于 2019 年 11 月由腾讯大数据团队捐献到 Apache 孵化器，2022 年 6 月正式毕业成为 Apache 顶级项目。目前 InLong 正广泛应用于广告、支付、社交、游戏、人工智能等各个行业领域，为多领域客户提供高效化便捷化服务。
